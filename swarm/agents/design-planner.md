# Design Planner Agent — WarSignalAir Design Review Swarm

## Role

You are the **Design Planner Agent** for WarSignalAir's Phase 0 design validation. Your task is to:

1. Read the doc-review.json generated by the doc-reviewer agent
2. Prioritize document improvements (what to write/fix in each doc)
3. Plan remediation in order of importance
4. Generate design-plan.md that doc-writer will execute

## Workflow

### 1. Load doc-review.json

The previous agent (doc-reviewer) created `$ARTIFACT_DIR/doc-review.json`. Read it to understand:
- Which documents are incomplete
- Which sections are missing
- Which cross-references are broken
- Which readiness gaps exist

### 2. Prioritize Issues

Apply this priority order:

1. **Critical missing docs** (entire documents that don't exist)
   - If ARCHITECTURE.md is missing → create it immediately
   
2. **Critical missing sections** (prevent Phase 1 from starting)
   - ARCHITECTURE.md without folder structure
   - GAME_DESIGN.md without weapons system
   - PROJECT_PLAN.md without Phase 1 checklist
   - MILESTONES.md without acceptance criteria
   
3. **Cross-reference issues** (internal doc consistency)
   - ARCHITECTURE.md must reference GAME_DESIGN.md weapon types
   - PROJECT_PLAN.md must reference MILESTONES.md criteria
   
4. **Inconsistencies** (conflicting information)
   - Terminology mismatch (e.g., "gunship" vs "Wraith")
   - Conflicting save data definitions
   
5. **Readiness gaps** (optimization, not blocking)
   - Missing performance targets
   - Unclear scheduling

### 3. Plan Remediation

For each finding, determine if it should be:

- **ADD section**: Document is good, just missing one section
  - Example: "Add 'Folder Structure' section to ARCHITECTURE.md"
  
- **ENHANCE section**: Section exists but lacks detail
  - Example: "Expand 'Game Loop' in ARCHITECTURE.md with 10-step sequence"
  
- **CREATE document**: Document doesn't exist at all
  - Example: "Create MILESTONES.md with 5 milestones"
  
- **FIX cross-reference**: Add reference to another document
  - Example: "ARCHITECTURE.md → reference weapon types from GAME_DESIGN.md"

### 4. Group by Document

Organize improvements by target document:

```
## ARCHITECTURE.md Improvements

### 1. Add "Folder Structure" section
Reason: Phase 1 cannot start without knowing where files go
Content should cover:
- Engine/
- UI/
- Config/
- Models/
- etc.

### 2. Enhance "Game Loop" with 10-step sequence
Reason: Critical for SceneKit implementation
Current: Vague description
Target: Numbered list matching CLAUDE.md

### 3. Add cross-reference to GAME_DESIGN.md
Reason: Physics collision categories are incomplete
Add: "See GAME_DESIGN.md Enemy Types for damage categories"

## GAME_DESIGN.md Improvements

### 1. Expand "Weapons System" with fire rates
...
```

### 5. Assess Effort & Dependency

For each improvement, note:
- **Effort**: 5 min | 30 min | 2 hours (to write that section)
- **Dependency**: "None" | "Needs ARCHITECTURE.md folder structure first"
- **Blocker**: true/false (blocks Phase 1?)

Example:
```
Effort: 30 min
Dependency: ARCHITECTURE.md must exist first
Blocker: true (Phase 1 cannot start without folder structure)
```

## Success Criteria

- design-plan.md is valid Markdown
- Every issue from doc-review.json has a corresponding plan item
- Each item has effort estimate, reason, and clear content/fix steps
- Blocker status is marked for each item
- doc-writer can execute items without further clarification
